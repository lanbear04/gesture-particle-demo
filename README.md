# 手势粒子交互系统

[![在线体验](https://img.shields.io/badge/🚀%20在线体验-点击访问-blue)](https://你的-vercel-或-github-pages-链接)

基于摄像头的单手手势粒子特效。用食指指尖隔空"画"出粒子，握拳变色，一键切换爆炸 / 旋涡 / 爱心 / 穿梭四种粒子模式；画面默认不额外显示食指指尖追踪点，方便沉浸展示。

- 爱心模式：粒子聚成一颗会心跳的心，捏合控制聚散，五指张开炸成漫天碎心。
- 穿梭模式：进入真 3D 视角，观察外层空心 3D 爱心 + 内层实心 3D 爱心；握拳拉近、五指张开推远、普通手型保持距离，无手自动展示，比 1 控制水平环绕、双指并拢左右移动控制上下环绕。

## 技术栈

- **手势识别**：[MediaPipe Tasks Vision](https://ai.google.dev/edge/mediapipe) HandLandmarker（浏览器端，21 点，数据不出本地）
- **渲染**：Canvas 2D + 自写粒子引擎（`lighter` 发光合成 + 半透明覆盖拖尾）；穿梭模式在 Canvas2D 中实现 3D 透视投影与景深近似
- **工程**：Vite + TypeScript，无前端框架

WASM 运行时与模型文件均在 `public/` 本地托管，不依赖 CDN，离线可用。

## 运行

```bash
npm install      # 安装依赖（已包含 MediaPipe WASM）
npm run dev      # 启动开发服务器（localhost，自动打开浏览器）
```

打开后浏览器会请求摄像头权限，点击「允许」。建议在光线充足处使用。

其他命令：

```bash
npm run build      # 类型检查 + 生产构建到 dist/
npm run typecheck  # 仅类型检查
npm run preview    # 预览生产构建
```

> 注意：摄像头需要安全上下文。`localhost` 已满足；若要在局域网其他设备访问，需配置 HTTPS。

## 操作

| 操作 | 效果 |
|------|------|
| 移动食指 | 指尖位置持续发射粒子 |
| 握拳 ✊ | 爆炸/旋涡模式下粒子变红（松开恢复青蓝） |
| 点「爆炸」/ 按 `1` | 切换到爆炸模式（向四周炸开） |
| 点「旋涡」/ 按 `2` | 切换到旋涡模式（围绕指尖打旋） |
| 点「爱心 ♥」/ 按 `3` | 切换到爱心模式（粒子聚成心形） |
| 点「穿梭 🚀」/ 按 `4` | 切换到穿梭模式（3D 相机环绕外层空心爱心 + 内层实心爱心） |
| 爱心模式下捏合 🤌 | 粒子被吸向心形轮廓（捏得越紧聚得越实） |
| 爱心模式下张开 🖐️ | 几乎无吸力，粒子自然散开漂移 |
| 爱心模式下五指张开 🖐️✨ | 爱心「砰」地炸成漫天碎心（上升沿触发，有冷却） |
| 穿梭模式下握拳 | 相机拉近心形 |
| 穿梭模式下五指张开 | 相机推远心形 |
| 穿梭模式下普通手型 | 保持当前相机距离 |
| 穿梭模式下手移出画面 | 触发缓慢自动旋转展示 |
| 穿梭模式下手指比 1 | 左右移动食指，控制水平环绕；不改变远近 |
| 穿梭模式下食指+中指并拢 | 左右移动双指，控制上下环绕；不改变远近 |
| 点「Params」/ 按 `T` | 打开穿梭模式调参面板，可实时调整结构、粒子密度、残影、相机距离等 |
| 调参面板点「Save default」 | 将当前穿梭参数保存到浏览器本地，刷新后自动作为默认值 |
| 点「镜头」/ 按 `C` | 切换摄像头画面显示（关闭 / 淡显 / 清晰 三档循环） |
| 点「关键点」/ 按 `H` | 切换手部 21 点骨架可视化（默认关闭；打开后绿色骨架，指尖标红，仅用于调试） |
| 点「全屏」/ 按 `F` | 进入或退出全屏展示模式，隐藏按钮、状态、FPS/阈值和调参面板，并临时关闭关键点骨架 |

左上角实时显示 FPS、四指伸展比（调试握拳阈值用）。爱心模式下会显示当前捏合程度；穿梭模式下会显示当前远近控制值，底部状态会显示「握拳：拉近」「五指张开：推远」「保持距离」「无手：自动展示」「比1：水平环绕」「双指：上下环绕」等识别结果。

全屏展示模式适合演示和录屏：界面只保留摄像头背景与粒子画面，不显示控制按钮、左上角调试信息、底部状态、关键点骨架或调参面板；退出全屏后会恢复进入前的关键点骨架开关状态。

## 调参

调参集中在 [src/config.ts](src/config.ts)：

- `PARTICLES.*`：粒子池上限、发射速率、阻尼、半径、生命。
- `TRAIL_FADE_ALPHA`：拖尾长度（越小越长）。
- `COLORS.*`：默认色、握拳色、插值速度。
- `FIST.curlRatio` / `FIST.debounceFrames`：握拳灵敏度与去抖。
- `HEART.*`：爱心模式采样点数、心形大小、吸力范围、脉动节奏、爆发冲量、冲击波参数。
- `HEART_COLORS.*`：爱心模式松散/紧实颜色。
- `SHUTTLE.*`：穿梭模式 3D 心形规模、外层空心壳厚度、内层实心爱心比例、相机环绕半径范围、景深虚化参数、拖尾和手势识别阈值。

穿梭模式也提供运行时调参面板：

- `Heart shape`：外层大小/厚度、外层空心壳厚、内层大小/厚度。
- `Particle density`：粒子总量、内层占比、外/内层粒径。
- `Motion`：残影长度、自动旋转速度。
- `Camera`：最远相机距离。
- `Save default` 会把当前穿梭参数保存到浏览器 `localStorage`，刷新后自动加载；`Reset` 会回到已保存默认值（若没有保存过，则回到代码内置默认值）。

## 当前范围（MVP）

已实现：单手 21 点追踪、指尖跟随 + 拖尾（不额外显示指尖追踪点，关键点骨架默认关闭）、爆炸 / 旋涡 / 爱心 / 穿梭四模式切换、握拳变色、爱心模式捏合聚散与五指张开爆开、穿梭模式真 3D 透视与多手势相机控制、运行时调参面板、本地保存穿梭默认参数、镜头显示开关、关键点骨架调试开关、全屏沉浸展示模式。

未做：双手交互、自定义粒子编辑器、复杂组合手势、录制回放、云端存储。

## 实现注记

- 渲染循环约 60fps 而摄像头约 30fps，`HandTracker.detect` 对视频帧未推进的情况返回 `"stale"`，由 `App` 保持上一帧手势状态——否则握拳的原始信号会逐帧闪烁，去抖永远无法累积。
- 粒子使用固定大小对象池，运行期不 `new`，避免 GC 抖动。
- 所有手势判定基于 2D 归一化坐标（不含 z），因为 MediaPipe 的 z 估计在正对摄像头握拳时不可靠，含 z 反而会让握拳判定失效。
- 穿梭模式采用隐式心形体拒绝采样：外层是大 3D 爱心减去内部缩小 3D 爱心得到的空心厚壳，内层是独立实心 3D 爱心；粒子通过位置插值收敛到目标点后保持静止，所有动态来自相机环绕。
- 穿梭模式手势识别使用专用规则：模式 4 的远近控制来自握拳/五指张开两个离散手势；五指张开使用比爱心爆发更宽松的阈值，允许小指等少量手指因角度偏短；比 1 和双指并拢仍使用独立判定；普通手型保持当前距离，并对手势模式做短帧数稳定确认，减少状态误跳。
- 双指并拢不是靠一个硬阈值判断：会计算综合评分（两指距离、伸展比、直度、方向平行度、无名指/小指退出控制程度）。穿梭远近控制不依赖手靠近镜头、C 形松手或捏合，减少识别丢失和手势互抢的问题。
- 穿梭模式相机控制使用当前帧原始手势即时驱动，UI 文案使用稳定后的手势状态显示；比 1 / 双指并拢出现时会立刻 `freezeRadius()`，保持当前相机远近不变。
- 穿梭模式的景深虚化在 Canvas2D 中用透明度和尺寸衰减近似模拟（非真高斯模糊）。粒子按深度后到前排序，配合加色发光（`lighter`）营造体积光感。后续粒子量增大可迁移到 WebGL 渲染器（`Renderer` 接口已抽象，上层无需改动）。
